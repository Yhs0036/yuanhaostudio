# AI 使用说明

## 1. 使用了哪些 AI 工具

本项目主要使用了 Codex 作为 AI 辅助开发工具，用于需求拆解、代码生成、样式调整、问题排查、性能检查和部署流程辅助。在开发过程中，也参考了 React Bits 组件的集成说明，用于实现卡片边缘光效和首屏 WebGL 动态背景的视觉效果。

## 2. AI 帮助完成了哪些工作

- 根据笔试题要求和我的简历内容，规划个人网站的信息结构，包括首页 Hero、精选项目、能力模块和底部联系方式。
- 使用 React + Vite 从零搭建项目，并补充 `package.json`、`vite.config.js`、`README.md`、部署配置等文件。
- 将简历中的教育背景、项目经历和技能能力改写成更适合网页展示的文案。
- 实现暗色系、高级克制、偏科技感的页面风格，包括首屏背景、导航栏、项目卡片、能力卡片和联系方式区域。
- 集成并调整 React Bits 风格组件，包括卡片边缘光效和 Prism WebGL 动态背景。
- 帮助排查页面交互问题，例如按钮跳转、导航高亮等。
- 协助完成性能检查，重点检查了视频/图片体积、WebGL 背景、滚动监听、GSAP 动画残留和首屏加载。
- 配置 Vercel 部署文件。

## 3. 自己手动修改了哪些内容

- 决定网站的整体方向：暗色系、高级、克制、有科技感。
- 我提供了个人简历、排版参考截图和 React Bits 组件说明。
- 我多次根据预览效果提出修改意见，例如首屏排版、卡片左右顺序、导航悬浮磨砂质感、按钮功能、字体重叠问题等。
- 完成了 Vercel 账号登录、Vercel Drop 上传、GitHub 仓库创建、代码推送和 Vercel 连接 GitHub 仓库等需要授权的操作。
- 我最终确认了网站名称、线上部署结果和提交链接。

## 4. 遇到的问题以及如何通过 AI 解决

- 首屏排版早期存在空白区域和文字重叠问题。通过调整布局结构、卡片位置、文字宽度和响应式约束后解决。
- 导航栏最初只能作为静态链接使用。通过增加滚动监听和当前模块识别，实现滚动到不同模块时自动高亮。
- 顶部“联系我”按钮和联系方式按钮最初功能不明确。通过 AI 辅助改为跳转到底部联系区、复制邮箱/电话/微信等明确交互。
- 性能检查时发现旧的视频资源 `hero-loop.mp4` 体积约 3.3MB 且未被使用。通过 AI 辅助删除该文件，减少项目体积。
- WebGL 动态背景可能增加首屏负担。将 Prism 背景拆分为懒加载 chunk，降低 DPR 上限，减少 shader 步数，并在离开首屏或页面切到后台时暂停渲染。
